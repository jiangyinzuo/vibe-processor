# 昇腾 NPU 与 NVIDIA GPU 的设计思想：从第一性原理出发

## 1. 问题的起点

处理器设计的第一性问题不是“如何堆叠更多计算单元”，而是：

```text
给定一类主要工作负载，如何在功耗、面积、带宽和可编程性约束下，最大化有效吞吐？
```

不同芯片的设计思想，来自对这个问题中“主要工作负载”和“约束条件”的不同回答。

昇腾 NPU 和 NVIDIA GPU 都属于高吞吐并行处理器，但二者的历史起点不同：

- 昇腾 NPU 的核心目标是高效执行人工智能中的张量计算，尤其是矩阵乘、卷积、向量算子和围绕它们的数据搬运。
- NVIDIA GPU 的历史起点是实时图形渲染，即在严格帧时间内并行处理大量顶点、片元、纹理采样和像素输出；后来通过 CUDA 和通用计算生态扩展为通用并行处理平台，并进一步加入 Tensor Core 服务 AI 工作负载。

因此，二者不是同一个架构问题的两个实现细节，而是从不同目标函数出发形成的两种体系结构路线。

## 2. 高吞吐计算的共同基础

无论 NPU 还是 GPU，高吞吐计算都面对三个基本事实。

第一，现代芯片的算术单元很便宜，数据移动很昂贵。一次乘加的能量和面积代价通常远小于从片外存储器搬运操作数的代价。因此，高性能计算芯片必须减少片外访问，并尽量复用片上数据。

第二，单线程低延迟不是主要目标。AI 和图形工作负载通常包含大量相似操作，适合用并行性换取吞吐。芯片设计应优先让许多计算同时推进，而不是让单个指令流以最低延迟完成。

第三，硬件不能只追求峰值算力。真正有意义的是有效算力，即计算单元在真实程序中持续获得数据、持续发射操作、持续写回结果的能力。

由此得到高吞吐芯片的共同设计原则：

```text
并行执行 + 局部数据复用 + 高带宽存储层次 + 明确的调度机制
```

昇腾 NPU 和 NVIDIA GPU 的差异，主要体现在它们如何实现这四项原则。

## 3. 昇腾 NPU：面向张量数据流的专用加速器

### 3.1 设计目的

昇腾 NPU 的设计目标可以概括为：以较高能效执行神经网络中的规则张量计算。

神经网络中的核心计算往往可以规约为矩阵乘或卷积。卷积经过 im2col、tiling 或专用数据布局后，也可以表现为高度规则的乘加阵列计算。此类工作负载具有几个特征：

- 操作类型相对集中，矩阵乘和向量后处理占据主要时间。
- 数据访问模式在编译期通常可分析。
- 计算密度高，但片外带宽仍可能成为瓶颈。
- 性能依赖 tile 划分、片上缓存复用和搬运计算重叠。

因此，NPU 的自然设计方向不是提供最通用的线程机器，而是构造一条围绕张量 tile 的数据流执行管线。

### 3.2 从矩阵乘推导架构

考虑矩阵乘：

```text
C = A × B
```

每个输出元素需要对一行 A 和一列 B 做多次乘加。若每次乘加都从片外读取 A 和 B，带宽需求会迅速超过可承受范围。第一性推导得到的关键策略是：把矩阵切成 tile，并让每个 tile 在片上被重复使用。

因此，NPU 需要：

1. 片外或共享存储保存大矩阵。
2. 片上缓冲保存当前 tile。
3. 专用矩阵计算阵列持续消费 tile。
4. 数据搬运单元提前搬入下一批 tile，并搬出上一批结果。
5. 同步机制保证计算只在数据 ready 后发生。

这正是昇腾式 AI Core 中 Cube、Vector、Scalar、MTE 和多级片上存储的基本逻辑。

### 3.3 Cube 的意义

Cube 单元的本质是矩阵乘专用执行部件。它通常围绕固定形状的 tile 工作，例如把 L0A、L0B 中的数据送入矩阵乘阵列，并把结果写入 L0C。

这种设计牺牲了一部分通用性，换来三个收益：

- 计算阵列结构规则，面积和功耗效率高。
- 数据通路固定，便于高吞吐流水。
- 编译器可以围绕 tile、buffer 和 event 做静态调度。

Cube 不是孤立的乘法器集合。只有当 L0A/L0B 能稳定供数、L0C 能保存中间结果、MTE 能及时搬运数据时，Cube 的峰值算力才可能转化为有效吞吐。

### 3.4 MTE 和显式数据流

昇腾式 NPU 的重要思想是：数据搬运是一等公民。

MTE 负责不同存储层次之间的数据移动，例如从全局存储到 UB，从 UB 到 L0A/L0B，从 L0C 搬出结果。软件或编译器需要显式安排：

```text
CopyIn  ->  Compute  ->  CopyOut
```

这与依赖大 cache 自动掩盖访存不同。NPU 假定程序能够以 tile 为单位预测数据需求，因此把更多责任交给编译器和运行时：

- 哪个 tile 先搬入。
- 哪段 UB 可以复用。
- 哪个 event 表示数据已经 ready。
- 哪个计算可以与哪次搬运重叠。

这种设计的核心优点是确定性。只要调度正确，硬件可以按规则高效推进；核心代价是编程模型和编译器更复杂。

### 3.5 NPU 的设计思想总结

昇腾 NPU 的体系结构思想可以概括为：

```text
用专用张量执行阵列提高单位能耗算力，
用显式片上存储层次减少片外访问，
用 MTE 和 event/token 把数据搬运编排成可预测流水。
```

它追求的是张量工作负载上的能效和确定性吞吐，而不是任意控制流程序上的最大灵活性。

## 4. NVIDIA GPU：面向大规模线程并行的通用吞吐处理器

### 4.1 设计目的

NVIDIA GPU 的原始目标是实时图形渲染。图形渲染的工作负载与神经网络张量计算不同。它要在每一帧内处理大量顶点、三角形、片元和像素，每个元素的处理相似但不完全相同，并且包含纹理访问、插值、分支、深度测试和混合等操作。

这类工作负载具有几个特征：

- 并行元素数量极大。
- 单个元素的控制流和访存可能不完全规则。
- 延迟必须被隐藏，否则帧时间无法满足要求。
- 可编程性重要，因为图形 shader 需要表达多样化效果。

因此，GPU 的自然设计方向是：提供大量轻量线程，并用硬件调度在这些线程之间切换，以隐藏内存和执行延迟。

### 4.2 从图形渲染推导架构

实时渲染要求在固定时间内完成大量类似任务。每个像素或片元可以看作一个小程序实例。如果某个实例等待内存，硬件不应空转，而应切换到另一个 ready 实例。

由此推导出 GPU 的基本机制：

1. 把程序组织为大量线程。
2. 把线程分组为 warp，使用 SIMT 方式执行。
3. 每个 SM 保持多个 warp 常驻。
4. Warp scheduler 每周期选择 ready warp 发射。
5. 用多 warp 并发隐藏访存和长延迟操作。

GPU 的核心不是单个线程很快，而是系统中总有其他线程可以运行。

### 4.3 SIMT 的意义

SIMT 可以理解为“以线程形式编程，以向量方式执行”。程序员看到的是线程，硬件执行时把一组线程组成 warp，在同一条指令下并行处理多个 lane。

这带来两个重要结果：

- 对规则数据并行程序，SIMT 可以高效利用硬件。
- 对分支发散程序，同一个 warp 内不同 lane 需要按分支路径分批执行，效率下降。

因此，GPU 的设计思想不是完全消除不规则性，而是通过大量线程和硬件调度容忍不规则性。

### 4.4 存储层次与延迟隐藏

GPU 也重视数据复用，但它采用的机制更偏通用处理器路线：

- Global memory 提供大容量高带宽。
- L2 和各级 cache 缓解访存压力。
- Shared memory 为程序员提供显式片上缓存。
- Register file 支撑大量常驻线程。
- Warp scheduler 用 ready warp 覆盖等待周期。

与 NPU 相比，GPU 不要求所有数据搬运都被显式声明为 tile pipeline。程序可以依赖 cache、coalescing、shared memory 和 warp 调度共同提高有效带宽。

这使 GPU 更灵活，能运行图形、科学计算、物理仿真、传统 GPGPU 算法和 AI 算子。但代价是：对高度规则的矩阵乘，如果不引入 Tensor Core 这类专用单元，纯标量/SIMT 执行的能效通常不如专用张量阵列。

### 4.5 Tensor Core 的位置

现代 NVIDIA GPU 增加 Tensor Core，是因为 AI 工作负载的重要性已经足以改变 GPU 内部资源配置。Tensor Core 可以看作 GPU 在保持通用 SIMT 编程模型的同时，加入专用矩阵执行部件。

这说明 GPU 的设计路线不是拒绝专用化，而是在通用吞吐处理器框架内逐步吸收专用单元：

```text
通用线程并行框架 + 专用矩阵执行单元 + 硬件调度和缓存体系
```

因此，现代 GPU 与 NPU 在 AI 计算上出现了一定收敛：二者都需要矩阵阵列和片上复用。但它们的外层控制哲学仍然不同。GPU 以线程和 warp 为基本抽象；NPU 更强调 tile、buffer、MTE 和 event。

### 4.6 GPU 的设计思想总结

NVIDIA GPU 的体系结构思想可以概括为：

```text
用大量线程暴露并行性，
用 SIMT 批量执行相同指令，
用硬件 warp 调度隐藏延迟，
用缓存和 shared memory 兼顾通用性与带宽效率。
```

它追求的是广泛并行工作负载上的高吞吐和可编程性，而不是只针对单一数据流形式获得最高确定性能效。

## 5. 两种设计路线的对照

| 维度 | 昇腾 NPU | NVIDIA GPU |
|---|---|---|
| 初始目标 | AI 张量计算加速 | 实时图形渲染，后扩展到通用并行计算 |
| 基本抽象 | tile、buffer、dataflow、event | thread、warp、block、SM |
| 主要执行单元 | Cube/Vector/Scalar 等异构单元 | CUDA core、SFU、Tensor Core、load/store 等 |
| 并行方式 | tile 级数据流并行和矩阵阵列并行 | 大量线程和 warp 级 SIMT 并行 |
| 访存思想 | 软件/编译器显式管理片上存储和搬运 | cache、coalescing、shared memory、warp 调度共同隐藏延迟 |
| 同步方式 | event/token、queue、显式 producer/consumer 边界 | block/warp 同步、memory ordering、调度器 ready 选择 |
| 优势 | 规则张量计算能效高，数据流确定 | 通用性强，能容忍更复杂控制流和访存模式 |
| 代价 | 编程和编译器调度复杂，通用性较弱 | 控制和调度硬件复杂，规则矩阵计算需专用单元提高能效 |

## 6. 从第一性原理得到的结论

如果目标工作负载是高度规则的张量计算，则最自然的架构是：

```text
固定 tile 计算阵列 + 显式片上存储 + 显式数据搬运 + event/token 同步
```

这就是昇腾 NPU 的设计方向。

如果目标工作负载是大量相似但可能存在分支和不规则访存的并行程序，则最自然的架构是：

```text
大量轻量线程 + SIMT 执行 + 多 warp 常驻 + 硬件调度隐藏延迟
```

这就是 NVIDIA GPU 的设计方向。

二者的差异不是简单的“专用”和“通用”之分，而是对计算、存储和调度责任划分的不同选择：

- NPU 把更多调度责任交给编译器和显式数据流，以换取规则张量计算的能效。
- GPU 把更多调度责任放在硬件和线程运行时中，以换取更大的程序覆盖面和延迟容忍能力。

## 7. 对学习体系结构的启示

理解这两类芯片时，不应从模块名称开始，而应从目标函数开始：

1. 工作负载是否规则。
2. 数据是否可以被静态分块和复用。
3. 延迟是靠显式流水隐藏，还是靠大量线程隐藏。
4. 编译器、程序员和硬件各自承担多少调度责任。
5. 峰值算力能否被数据通路持续喂饱。

从这个角度看，昇腾 NPU 和 NVIDIA GPU 代表了两种成熟的高吞吐处理器思想。NPU 是以数据流和张量 tile 为中心的专用化路线；GPU 是以线程并行和硬件调度为中心的通用吞吐路线。现代 AI 计算的发展使二者在矩阵执行单元上趋近，但它们关于“谁来管理数据、谁来隐藏延迟、谁来承担调度复杂性”的根本答案仍然不同。
