# 昇腾NPU和Nvidia GPU

## 设计起源

昇腾 NPU 和 NVIDIA GPU 都属于高吞吐并行处理器，但设计出发点不同：

**NPU将矩阵运算当一等公民**。NPU 从一开始就认为，今天 AI 计算的核心归根结底就是**矩阵乘加**，**显式数据搬运**设计都是为了喂饱矩阵计算单元。

**GPU将向量和多线程当一等公民**。NVIDIA GPU 起源于实时图形渲染，随后通过 CUDA 扩展为**通用并行计算**平台，后来被用于AI加速。Tile programming只是后来新加的第二层模型。

## 高吞吐并行计算

NPU 和 GPU 作为面向高吞吐并行计算任务设计的处理器， 面对以下几个问题：

1. 片上算术操作的单位代价通常低于数据移动，尤其低于片外存储访问。如何提高片上数据复用率，降低片外带宽压力？
2. 大量并行计算任务中，单个指令流的低延迟不是设计重点，如何提高整体吞吐量？
3. 峰值算力不等于有效算力。如何设计数据搬运机制，喂饱计算单元？
