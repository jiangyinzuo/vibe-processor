# DMA-Compute Overlap 性能测量报告

## 测试环境

- **NPU 配置：** 1 个 AiCore，8×8 收缩阵列
- **HBM 延迟：** 10 周期
- **测试数据：** 3 个 8×8 矩阵乘法 tile
- **测试日期：** 2026-05-01

---

## 测试方法

### 基准测试：顺序执行（无 Overlap）

**程序结构：**
```
for each tile:
  DMA_LOAD A
  DMA_LOAD W
  DMA_WAIT          ← 等待 DMA 完成
  LOAD A -> L0
  LOAD W -> L0
  MATMUL            ← 计算
  STORE C -> UB
  DMA_STORE C
  DMA_WAIT          ← 等待 DMA 完成
```

**特点：**
- DMA 和计算完全串行
- 每个 tile 独立完成后再处理下一个
- 无重叠

### 优化测试：流水线 Overlap

**程序结构：**
```
// Tile 0: 初始加载
DMA_LOAD A0
DMA_LOAD W0
DMA_WAIT
LOAD A0 -> L0
LOAD W0 -> L0

for tile 1..N:
  DMA_LOAD A[i]     ← 预取下一个 tile（非阻塞）
  DMA_LOAD W[i]     ← 预取下一个 tile（非阻塞）
  MATMUL            ← 计算当前 tile（与 DMA 重叠！）
  STORE C -> UB
  DMA_STORE C
  DMA_WAIT          ← 等待预取完成
  LOAD A[i] -> L0
  LOAD W[i] -> L0

// 最后一个 tile
MATMUL
STORE C -> UB
DMA_STORE C
DMA_WAIT
```

**特点：**
- 在计算 tile i 时预取 tile i+1
- DMA 和 MATMUL 并行执行
- 流水线重叠

## 测试结果

### 原始数据

| 指标 | 顺序执行 | 流水线 Overlap | 差异 |
|------|---------|---------------|------|
| 总周期数 | 557 | 455 | -102 (-18.3%) |
| Cube 计算周期 | 45 | 45 | 0 (相同) |
| DMA 周期 | 216 | 216 | 0 (相同) |
| 重叠周期 | 0 | 52 | +52 |
| 气泡周期 | - | - | - |
| 计算效率 | 8.1% | 9.9% | +1.8% |
| DMA 占比 | 38.8% | 47.5% | +8.7% |
| 重叠率 | 0.0% | 24.1% | +24.1% |
| 每个 Tile 平均周期 | 185.7 | 151.7 | -34.0 (-18.3%) |

### 性能提升

**实际加速比：**
```
加速比 = 557 / 455 = 1.22×
```

**周期节省：**
```
节省周期 = 557 - 455 = 102 周期
节省比例 = 102 / 557 = 18.3%
```

**重叠效果：**
```
重叠周期 = 52
重叠率 = 52 / 216 = 24.1%
```

这意味着有 24.1% 的 DMA 时间被计算隐藏了。

## 性能分析

### 为什么加速比是 1.22× 而不是理论的 1.42×？

**理论模型假设：**
- MATMUL 周期：15
- DMA 周期（每个 tile）：36
- 完全重叠

**实际情况：**
1. **MATMUL 周期更长：** 实际 45 / 3 = 15 周期（符合理论）
2. **DMA 周期更长：** 实际 216 / 3 = 72 周期（理论 36）
   - 原因：HBM 延迟 10 周期 + 数据传输时间
3. **部分重叠：** 只有 24.1% 的 DMA 被隐藏
   - 原因：DMA 时间 (72) > MATMUL 时间 (15)
   - MATMUL 无法完全隐藏 DMA

### 重叠率分析

**为什么重叠率只有 24.1%？**

```
每个 tile 的时间分解：

顺序执行：
  DMA_LOAD:  72 周期
  LOAD:      ~10 周期
  MATMUL:    15 周期
  STORE:     ~10 周期
  DMA_STORE: 72 周期
  其他:      ~6 周期
  总计:      ~185 周期

流水线 Overlap：
  Tile 0 (初始):
    DMA_LOAD:  72 周期
    LOAD:      ~10 周期
  
  Tile 1-2 (流水线):
    DMA_LOAD (并行): 72 周期
    MATMUL (并行):   15 周期  ← 只有这 15 周期与 DMA 重叠
    STORE:           ~10 周期
    DMA_STORE:       72 周期
    LOAD:            ~10 周期
  
  重叠周期 = 15 × 2 = 30 周期（理论）
  实际测量 = 52 周期（包含部分 STORE 时间）
```

**瓶颈：**
- DMA 时间 (72) >> MATMUL 时间 (15)
- MATMUL 只能隐藏 15/72 = 20.8% 的 DMA 时间
- 实际测量 24.1% 说明还隐藏了部分 STORE 时间

### 计算效率提升

**顺序执行：**
```
计算效率 = 45 / 557 = 8.1%
```

**流水线 Overlap：**
```
计算效率 = 45 / 455 = 9.9%
```

**提升原因：**
- 总周期减少，但计算周期不变
- 更多时间用于实际计算

## 优化空间

### 当前限制

1. **DMA 时间过长**
   - 当前：72 周期/tile
   - 瓶颈：HBM 延迟 + 数据传输
   - 改进：增加 L2 缓存，减少 HBM 访问

2. **MATMUL 时间过短**
   - 当前：15 周期
   - 问题：无法完全隐藏 DMA
   - 改进：增大 tile 尺寸（8×8 → 16×16）

3. **部分重叠**
   - 当前：24.1%
   - 理论最大：~100%（当 MATMUL ≥ DMA）
   - 改进：优化流水线结构

### 优化建议

#### 1. 增大 Tile 尺寸

**当前：** 8×8 tile
- MATMUL 周期：15
- DMA 周期：72
- 比例：1:4.8

**优化：** 16×16 tile
- MATMUL 周期：~60 (4× 增长)
- DMA 周期：~144 (2× 增长)
- 比例：1:2.4
- **预期重叠率：~40%**

#### 2. 增加 L2 缓存

**当前：** L2 深度 2048
- 需要频繁访问 HBM
- HBM 延迟 10 周期

**优化：** L2 深度 8192
- 减少 HBM 访问
- DMA 周期减少 ~30%
- **预期加速比：1.35×**

#### 3. 多级流水线

**当前：** 2 级流水线（预取 + 计算）
- Tile i: 计算
- Tile i+1: 预取

**优化：** 3 级流水线（预取 + 计算 + 写回）
- Tile i: 写回
- Tile i+1: 计算
- Tile i+2: 预取
- **预期重叠率：~50%**

#### 4. 双缓冲优化

**当前：** LOAD 完成后立即切换 buffer
- 无法在 MATMUL 期间预取

**优化：** MATMUL 开始时切换 buffer
- 允许在计算期间预取到 fill buffer
- **预期重叠率：+10%**

## 理论性能上限

### 最优情况分析

**假设：**
- Tile 尺寸：16×16
- L2 缓存：8192
- 3 级流水线
- 双缓冲优化

**预期性能：**
```
MATMUL 周期：60
DMA 周期：50 (优化后)
重叠率：100% (MATMUL > DMA)

顺序执行：
  总周期 = N × (DMA + MATMUL) = N × 110

流水线 Overlap：
  总周期 = DMA_initial + N × MATMUL = 50 + N × 60

加速比 = 110N / (50 + 60N) ≈ 1.83× (N 较大时)
```
